# OOPS v0.2.0 发布信息

## 🚀 重大架构升级

OOPS v0.2.0 是一个重要的架构升级版本，重新设计了检测流程和数据处理架构，为未来的功能扩展奠定了坚实基础。

## 🎯 核心改进

### 1. 全新检测顺序 🔄
```
硬件信息 → 系统信息 → 系统设置 → 网络连通性 → Python环境 → 组件依赖 → 路径规范
```

这个新顺序遵循"从底层到应用层"的逻辑，让问题诊断更加系统化和高效。

### 2. 增强 Python 环境检测 🐍

- **多包管理器支持**: 自动识别 pip、conda、uv、poetry、pipenv
- **智能虚拟环境检测**: 支持任意命名的虚拟环境
- **完整依赖分析**: 检测缺失包、版本冲突、依赖完整性
- **精准修复建议**: 提供具体的安装命令和解决方案

### 3. 数据渲染分离 🎨

- **纯数据展示**: 系统信息只显示数据，不显示警告
- **独立验证逻辑**: 硬件适配检测独立为检测项
- **统一数据格式**: 所有检测器遵循统一的数据模型

## 🔧 技术亮点

### 新增检测器
- `hardware.py` - 纯硬件信息收集
- `system.py` - 系统基本信息
- `system_settings.py` - 系统设置验证
- `python_environment.py` - Python 环境完整检测

### 架构改进
- 保持 100% 向后兼容
- 渐进式重构，每个阶段独立可回滚
- 更好的错误处理和日志记录

## 🎉 下载和使用

### 获取方式
- GitHub Releases 页面下载
- 支持 Windows 10/11
- 单文件可执行，无需安装

### 使用方法
```bash
# 基本检测
oops.exe

# 指定游戏配置
oops.exe --config configs/zenless_zone_zero.yaml

# 生成详细报告
oops.exe --output reports/
```
